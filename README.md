## Overview
This project is a responsive data storage component that displays how much storage has been used and how much is left. The component adapts to different screen sizes and includes a customized progress bar, decorative elements, and a clean, modern UI inspired by the original Fylo design.

The goal was to replicate the design as accurately as possible while keeping the code clean, readable, and maintainable.

## My process
- Structured the layout using **semantic HTML5** elements.
- Used **Flexbox** to align and distribute content within the main components.
- Styled the project using **CSS custom properties (variables)** to maintain a consistent color palette and reuse gradients.
- Customized the native `<progress>` element to match the design, including browser-specific styling.
- Applied **absolute positioning** for decorative elements like the storage indicator and triangle shape.
- Implemented **responsive design** using media queries to adapt the layout for desktop and mobile screens.
- Followed the **BEM methodology** for class naming to keep the CSS organized and scalable.

## What I learned
- How to customize native HTML elements like `<progress>` for a polished UI.
- Managing layouts with **Flexbox** while keeping components modular and reusable.
- Using **CSS variables** effectively for colors, gradients, and theme consistency.
- Creating decorative UI elements using pure CSS (`border-radius`, `clip-path`, `position`).
- Improving responsive layouts through media queries and adaptive positioning.
- Writing cleaner and more maintainable CSS by following **BEM conventions**.
